# /resolve-parallel

并行处理多个独立问题，提高效率。

## 流程

1. **解析问题** - 识别所有待解决的问题
2. **依赖分析** - 确定哪些问题可以并行处理
3. **并行执行** - 同时处理独立的问题
4. **汇总结果** - 报告处理结果

## 适用场景

- 批量修复 lint 错误
- 解决多个独立的 TODO
- 处理多个不相关的 bug

## 执行规则

- 只并行处理**无依赖**的问题
- 有依赖的问题按顺序处理
- 每个问题完成后立即报告

## 输出

- 已解决的问题列表
- 未能解决的问题及原因
- 建议的后续步骤
