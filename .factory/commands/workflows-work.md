# /workflows-work

执行计划文件中的任务，系统化地完成实施工作。

## 流程

1. **加载计划** - 读取并解析计划文件
2. **任务分解** - 将计划拆分为可执行任务
3. **逐步执行** - 按顺序完成每个任务
4. **进度追踪** - 更新计划文件中的复选框

## 执行规则

1. 读取计划文件，理解整体目标
2. 识别所有待办任务（`- [ ]` 标记）
3. 按顺序执行每个任务
4. 完成后将 `- [ ]` 改为 `- [x]`
5. 遇到阻塞时，记录问题并继续下一个任务

## 工作原则

- **小步提交** - 每完成一个功能点就提交
- **测试驱动** - 先写测试，再写实现
- **保持简单** - 选择最简单的解决方案
- **及时沟通** - 遇到不确定的决策时询问用户

## 输出

- 更新后的计划文件（标记已完成任务）
- 实现的代码变更
- 提交记录

## 后续步骤

完成后运行 `/workflows-review` 进行代码审查
